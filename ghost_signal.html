Ghost Signal: A Stellaris Game

I conducted multiple UX tests and design reviews for our title Ghost Signal, a Sci-Fi VR action roguelite set in the Stellaris Universe.

In my tests, I focused on improving the tutorial and onboarding experience, balancing, UI, and the run feeling.

I identified issues and suggested solutions, discussed design ideas, as well as helped prioritize issues and summarize feedback from other sources.

RUNS AND BALANCING

Through several UX tests, ranging from simple feature tests to fully fledged runs, I identified issues and suggested solutions to improve the experience.

Top insights and solutions from the tests include:

  • Moving the ship during battle caused stress, confusion and negative feelings for players. Adding slo-mo while placing waypoints reduced the negative feelings
  • Removing the shield encouraged players to learn the movement system
  • Using the scanner for different actions was complicated. Removing the need to press several buttons fixed this
  • Players wanted a wider enemy variety earlier on
  • Players missed notifications. Adding audio cues solved part of the problems
  • Identified unnecessary steps in menu navigation and removed these
  • Identified unclear aspects of the UI and readability issues and suggested solutions
  • Players didn't notice that 3 research trees existed
  • Visually differenciating enemies was hard
  • Players often accidentally selected buttons
TUTORIAL

Learning from my earlier experiences with VR tutorials, I advocated for creating an interactive tutorial built like a level, and to add contextual tutorials during the run.

During the playtest, the tutorial was found to be very helpful in improving game understanding, but there were still some unclear aspects.

Some insighs and solutions include:

  • Missile scarcity is unclear. Fixed by adding reminders and information
  • Unclear when to move to next sector. Fixed by audio cue in later build
  • Movement tutorial needed to be repeated. Reminders were added to the first sector
  • Tutorial taught unwanted behaviors. Removed the issues in a later build
  • Identified what was not taught and needed clarification
KEY TAKEAWAYS
  • Players want to see buttons on a virtual controller while learning
  • Contextual tutorials work best
