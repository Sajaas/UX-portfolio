Case Study

CheckMate

We had 3 school days to create an educational app that will improve learning and add value for students between 9-25.

try the prototype
The Problem

A web platform or mobile app that will improve learning and add value for students between 9-25, and can compete with existing apps.

Research & Empathy
  • Assumptions through mapping PAINS & GAINS
  • Dot voted on problem statements
  • Created empathy map
  • Interview script & preparations
  • Interviewing target group
Visualization & Research

To define the problems we mapped the user PAINS & GAINS on a whiteboard, and sorted them into themes. This allowed us to visualize our assumptions, and formulate problem statements. From there, we prepared an Empathy Map, and interview questions.

User Research & Interviews

To confirm our assumptions we interview users from the target audience. It allowed us to remove false assumptions and prioritize the problems that affected several users.

Narrowing the scope
  • Insight sorting
  • Scope can hold three main features
  • Dot voting on problem statements
  • Formulating the chosen problem statements as user stories
User Stories

Once we sorted the insights from the interviews, we created our Scope-document and prioritized the problem statements. We figured we could handle 3 key problems, we formulated the top 3 problem statements as User Stories.

"As a student, I want to be able to follow my progress to feel motivated"

"As a student, I want to have a clear structure, so that I know what's expected of me"

"As a student, I want the possibility to choose how to learn, so that I can rationalize my studies"

Ideating & Prototyping
  • Sorted insights into themes
  • Identifying features
  • Gamification focus
  • User Journey Mapping
  • Wireframing
  • Low-fi clickable prototype with Marvel Pop
Identifying Features

We sorted the insights further into themes, and from there identified features that could address the User Stories. We had a clear gamification focus that allowed us to address the progress and motivation User Stories.

User Journey

To gain insight into how the user would interact with the product, we created a User Journey. This allowed us to see all the steps in the product, and create a cohesive prototype with a flow.

Testing & Feedback
  • Writing scenarios for user testing
  • Low-fi prototype testing
  • Insight sorting
Prototype Testing

We wrote realistic scenarios and observed the target audience when they interacted with the prototype. We gained feedback and validations on our assumptions, which we incorporated into the hi-fi prototype.

"Navigate to the current subject"

"What's the status of your math progress?"

Hi-fi Prototype & Iteration
  • Creating a Hi-fi prototype in Figma
  • Second round of user testing
  • Updated prototype according to feedback
Final result

Incorporating user feedback from earlier tests, we created a Hi-Fi prototype in Figma. After a final round of testing, we had a good flow, an easy-to-understand prototype that engaged the audience, and fulfilled the requirements.

Key Takeaways
  • Decide on team roles beforehand
  • Decide on processes and tools
  • Set time references and stick to a schedule
  • Simplify wireframes
  • Don't lose sight of the MVP
