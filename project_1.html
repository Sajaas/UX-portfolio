CheckMate

We had 3 school days to create an educational app that will improve learning and add value for students between 9-25.

The problem

A web platform or Mobile app that will improve learning and add value for students between 9-25, and can compete with existing apps.

Research & empathy
  • Assumptions through mapping PAINS & GAINS
  • Dot voted on problem statements
  • Created empathy map
  • Interview script & preparations
  • Interviewing target group
  • Took notes during other interviews
Narrowing the scope
  • Insight sorting
  • Scope can hold three main features
  • Dot voting on problem statements
  • Formulating the chosen problem statments as user stories

"As a student, I want to be able to follow my progress in order to feel motivated"

"As a student, I want to have a clear structure, so that I know what's expected of me"

"As a student, I want the possibility to choose how to learn, so that I can rationalise my studies"

Ideating & prototyping
  • Sorted insights into themes
  • Identifying features that solve the problem statements
  • Gamification focus
  • User Journey Mapping
  • Wireframing
  • Low-fi clickable prototype with Marvel Pop
Testing & feedback
  • Writing scenarios for user testing
  • Low-fi prototype testing
  • Insight sorting

"Navigate to the current subject"

"What's the status of your math progress?"

Hi-fi prototype & iteration
  • Creating a Hi-fi prototype in Figma
  • Incorporating user feedback from earlier tests
  • Second round of user testing
  • Updated prototype according to feedback
  • Prepare presentation
Presentation & reflection
  • I pre-recorded a walkthrough of the prototype
  • Focused on the brief, our problem statements, the process and challenges
  • Takeaway: decide on team roles beforehand
  • Takeaway: decide on processes and tools
  • Takeaway: set time references and stick to a schedule
  • Takeaway: simplify wireframes
  • Takeaway: don't lose sight of the MVP
